# Contributing

Contributions that improve accuracy, safety, architecture support, or documentation are welcome.

## Requirements

- Never modify executable bytes automatically.
- Separate high-confidence project changes from report-only findings.
- Do not assign unsupported function names automatically.
- Preserve user-created names and comments.
- Document false-positive risks.
- Include the Ghidra version, processor language, file format, and expected behavior in pull requests.
